This project demonstrates some uses of window.matchMedia("(max-resolution: ###dpi)") Sadly, this was only supported by a few versions of Firefox. Get Firefox 15 PC | Mac

For instance, by multiplying the dpi by the pixel height I get the size in inches. If I also do this with the width then use the pythagorean theorem I know that you are viewing this on a unknown size diagonal screen. (Please disconnect any additional screens as they may corrupt the results of the primary screen)

I find this interesting because it means I can derive the level of portability of the screen. A 2048×1536 screen at 264dpi (9.7" iPad3) is much more portable than a 2048×1536 screen at 96dpi (20.64" desktop monitor). I can also tell the difference between two screens with the same number of pixels (1920×1080). One is 21.5" monitor (102dpi) and other is a a 42" TV (52dpi). Given that the page is being viewed on a 42" TV I can style the page for a "ten foot experience"

I can also display the three huge coin images below at their actual size. ecommerce sites could use this technique to display products such as jewelry, gadgets, accessories, clothing or even scientific specimens.

I can also adjust link padding sizes on touch-enabled devices to match "ideal" finger sizes.
